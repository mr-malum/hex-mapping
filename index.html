Hex Mapping Prototype

Standalone prototype. Flat-top hexes, drag terrain, POI glyphs, snapped region borders, roads, rivers, JSON export, and CSV import.

Grid

Persistent world IDs use an origin of 300:300, so the map can eventually expand north/west without negative IDs. Randomize changes only base terrain/features and leaves POIs, roads, rivers, and borders alone.

View

Use the mouse wheel over the map to zoom. Pan with the Pan tool, or hold Shift and drag, or use middle/right mouse drag. Arrow keys also pan.

Tool

Base Terrain

Feature Overlay

The feature list is filtered by the selected base terrain, preventing nonsense combos like Ocean + Forest. Farmland is treated as a Grassland feature, not a separate biome.

POI Type

Region Border

Hover near a face, then click or drag to toggle snapped edge segments.

Road / River

Roads and rivers now draw one segment per click-pair. Click a start hex, then an end hex. After the segment is drawn, the tool resets and waits for a new start hex.

Import / Export

Use the legacy/swap option if the imported CSV appears transposed compared to the old map.

Import Notes

CSV import expects columns like Hex_ID, Terrain, and optionally Region_ID_Ref. It ignores Map_XY for now and treats Hex_ID as authoritative. The coordinate-order selector lets you migrate legacy row:column data into future X:Y / column:row IDs.
